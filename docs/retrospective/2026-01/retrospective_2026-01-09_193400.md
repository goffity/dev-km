## Context: Before

- **Problem**: เมื่อตอบ PR review comment ที่ต้อง defer จะตอบลอยๆ ว่า "added to backlog" โดยไม่มี tracking
- **Existing Behavior**: ไม่มี mechanism สำหรับ track งานที่ defer จาก review
- **Why Change**: งานที่ defer อาจถูกลืม ไม่มี reference กลับมา
- **Metrics**: 0% tracking สำหรับ deferred work

---

## Context: After

- **Solution**: สร้าง GitHub issue ก่อน defer แล้ว reply พร้อม issue reference
- **New Behavior**:
  1. สร้าง issue ด้วย `gh issue create`
  2. Reply ว่า "Created #X to track this work"
- **Improvements**:
  - ทุก defer มี tracking
  - Issue มี context จาก review comment
  - สามารถ `/focus` กลับมาทำได้
- **Metrics**: 100% tracking สำหรับ deferred work
