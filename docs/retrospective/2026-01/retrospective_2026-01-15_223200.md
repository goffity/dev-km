## Context: Before

- **Problem**: Multi-agent orchestration สร้าง 50-100 session files ต่อวัน ทำให้ข้อมูลกระจัดกระจาย
- **Existing Behavior**: Auto-capture สร้าง individual session files ไม่มีการรวมกลุ่ม
- **Why Change**: ต้องการลด file fragmentation และรวม related sessions เข้าด้วยกัน
- **Metrics**: 50-100 files/day ใน multi-agent setup

---

## Context: After

- **Solution**: สร้าง `/consolidate` command พร้อม 3-tier grouping algorithm
- **New Behavior**: Sessions ถูกจัดกลุ่มตาม issue, file overlap, และ time proximity
- **Improvements**:
  - Daily summary file แทน individual files
  - Smart grouping ลดความซ้ำซ้อน
  - Configurable thresholds
- **Metrics**: N files → 1 daily-summary.md

## Decisions & Rationale

| Decision | Options Considered | Chosen | Rationale |
|----------|-------------------|--------|-----------|
| Grouping Priority | Single criterion vs multi-tier | 3-tier (issue → files → time) | Captures different relationship types |
| Default Mode | Execute vs dry-run | Dry-run | Safety first, prevent accidental changes |
| Output Format | Replace files vs create summary | Create summary, keep originals | Preserve raw data for reference |
| Bash Compatibility | Bash 4+ vs Bash 3.2 | Bash 3.2 compatible | macOS ships with Bash 3.2 |

### Key Features

1. **3-tier Grouping Algorithm**
   - Priority 1: Same issue number (#123, PROJ-123)
   - Priority 2: File overlap >50%
   - Priority 3: Time proximity (30 min window)

2. **Safety Features**
   - Dry-run mode by default
   - Path validation for temp cleanup
   - Input validation for all arguments

3. **Output Format**
   - YAML frontmatter with metadata
   - Grouped sessions with collapsible content
   - Ungrouped sessions preserved
