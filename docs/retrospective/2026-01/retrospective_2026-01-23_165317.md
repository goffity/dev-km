## Lessons Learned

- Batching related issues in one session is efficient when they share context (same codebase area)
- jq can handle surprisingly complex text processing (markdown parsing) but regex support is limited
- `gh pr merge --merge` in a chained command doesn't always execute - need explicit separate call
- GitHub doesn't always auto-close issues with "Fixes #N, #M, #O" in a single commit - may need manual close
