## Context: Before

- **Problem**: เมื่อตอบ "Fixed!" ใน PR review comment ไม่มี commit hash ให้ reviewer trace
- **Existing Behavior**: Reply แค่ "Fixed!" โดยไม่ระบุว่าแก้ใน commit ไหน
- **Why Change**: Reviewer ต้องการ trace ว่าแก้ใน commit ไหน เพื่อ verify
- **Metrics**: 0% traceability

---

## Context: After

- **Solution**: เพิ่ม commit hash capture และใส่ใน reply
- **New Behavior**: Reply "Fixed in ce97912!" พร้อม commit hash
- **Improvements**:
  - Reviewer trace ได้ทันที
  - ใช้ quoted heredoc ป้องกัน shell injection
  - 3-step workflow ชัดเจน
- **Metrics**: 100% traceability

## Decisions & Rationale

| Decision | Options Considered | Chosen | Rationale |
|----------|-------------------|--------|-----------|
| Hash format | Full hash vs short hash | Short hash (`--short`) | อ่านง่าย, พอสำหรับ reference |
| Security | `<<EOF` vs `<<'EOF'` | `<<'EOF'` (quoted) | ป้องกัน shell injection จาก reviewer comment |
| Workflow | Inline vs 3-step | 3-step explicit | ชัดเจนว่าต้อง commit ก่อน capture hash |

## Lessons Learned

- **Always use `<<'EOF'` (quoted)** when heredoc content might contain untrusted data (like reviewer comments)
- **3-step workflow** makes complex operations clearer than inline
- **Same hash scenario** - when fixing multiple comments in one commit, all replies reference same hash
