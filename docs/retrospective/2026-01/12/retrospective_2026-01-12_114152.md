## Context: Before

- **Problem**: Reply ไปที่ PR review comments ไม่มี commit hash ระบุ
- **Existing Behavior**: Reply แค่บอก "Fixed!" โดยไม่บอกว่า fix ใน commit ไหน
- **Why Change**: Reviewer ไม่สามารถ trace ได้ว่า commit ไหนแก้ไข feedback ของตน
- **Code Pattern**: ใช้ quoted heredoc (`<<'EOF'`) กับ sed ที่ไม่ทำงาน (sed รันหลัง API call)

---

## Context: After

- **Solution**: ใช้ direct variable `${COMMIT_HASH}` ใน reply template
- **New Behavior**: Reply มี format `Fixed in abc1234! [description]`
- **Improvements**:
  - Reviewer trace commit ได้ง่าย
  - Step 9 เปลี่ยนจาก "Commit and Push" เป็น "Push" only
  - เพิ่มตัวอย่าง escaping สำหรับ special characters

## Lessons Learned

- `cmd | sed` ใน shell จะรัน sed กับ **output** ของ cmd ไม่ใช่ input
- Quoted heredoc (`<<'EOF'`) ป้องกัน variable expansion - ต้องใช้ unquoted ถ้าต้องการ expand
- `-F body=@file` ปลอดภัยกว่า `-f body="..."` สำหรับ arbitrary content
