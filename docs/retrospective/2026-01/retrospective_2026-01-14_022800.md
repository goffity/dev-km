## Context: Before

- **Problem**: `gh pr create` in `/pr` and `/td` commands didn't specify `--base` branch
- **Existing Behavior**: PRs were created targeting the default branch (main/master) automatically
- **Why Change**: Some projects need PRs to target `develop` branch instead of main
- **Metrics**: Workers in multi-agent system created PRs to wrong branch

---

## Context: After

- **Solution**: Added base branch detection logic before `gh pr create`
- **New Behavior**: Checks for `origin/develop` first, falls back to default branch
- **Improvements**: PRs now target the correct branch automatically
- **Metrics**: No more PRs to wrong branch

### Logic Added

```bash
# Detect base branch (prefer develop, fallback to default)
if git branch -r | grep -q "origin/develop"; then
  BASE_BRANCH="develop"
else
  BASE_BRANCH=$(git remote show origin | grep 'HEAD branch' | cut -d: -f2 | xargs)
fi
```
