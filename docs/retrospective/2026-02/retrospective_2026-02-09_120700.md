## Context: Before

- **Problem**: `jira-client.sh` `create` and `create-subtask` commands only supported `project`, `summary`, `description`, `issuetype` fields. No way to set labels or story points during issue creation.
- **Existing Behavior**: Users had to create issues first, then manually add labels and story points via separate API calls or Jira UI.
- **Why Change**: Discussion #100 feature request - users need to set labels and story points at creation time for workflow efficiency.
- **Metrics**: 4 fields in create payload, 0 optional metadata flags

---

## Context: After

- **Solution**: Added `--label`/`--labels` (comma-separated) and `--points` (numeric) flags to both `cmd_create()` and `cmd_create_subtask()`. Made story points field configurable via `JIRA_STORY_POINTS_FIELD`.
- **New Behavior**: Users can set labels and story points inline during issue creation.
- **Improvements**: One API call instead of multiple, configurable custom field ID, backward compatible.
- **Metrics**: 6+ fields in create payload, 2 new optional flags, labels auto-trimmed

## Decisions & Rationale

| Decision | Options Considered | Chosen | Rationale |
|----------|-------------------|--------|-----------|
| Flag naming | `--label` only vs `--labels` only vs both | Both `--label`/`--labels` | User convenience, consistency with existing `create-story` |
| Story points field | Hardcoded `customfield_10016` vs configurable | Configurable via `JIRA_STORY_POINTS_FIELD` | Different Jira instances may use different field IDs |
| Label handling | Simple split vs trim+filter | Trim whitespace + filter empties | Handles `"a, b,"` gracefully per Copilot review |
| Error message | Hardcoded flag name vs `$1` | Use `$1` | Shows the exact flag variant the user typed |

## Lessons Learned

- GitHub Discussions don't support auto-close via `Fixes #N` syntax - must use GraphQL `closeDiscussion` mutation manually
