## Context: Before

- **Problem**: เมื่อสร้าง Jira issues ต้อง format เองทุกครั้ง ไม่มี standard template
- **Existing Behavior**: `create` command สร้าง issue พื้นฐานเท่านั้น
- **Why Change**: ต้องการให้มี format ที่ครบถ้วนเพื่อให้ทีมทำงานต่อได้โดยไม่ต้องถามเพิ่มเติม
- **Metrics**: ไม่มี template commands

---

## Context: After

- **Solution**: เพิ่ม template functions และ commands ใหม่
- **New Behavior**: 
  - `create-story` และ `create-epic` สร้าง issue พร้อม template มาตรฐาน
  - `create-subtask-templated` สร้าง subtask พร้อม dependency links
  - `add-labels` เพิ่ม labels ให้ issue
  - `link` สร้าง issue links (blocks, blocked-by, relates)
- **Improvements**: 
  - Story/Epic template มี 7 sections: Overview, Requirements, System Flow, Subtask Dependencies, Acceptance Criteria, DoD, Test Scenarios
  - Subtask template มี 4 sections: Scope, Dependencies, Tasks, Acceptance Criteria
  - รองรับ auto-add labels และ dependency links
- **Metrics**: 5 new commands, 2 template functions, 709 lines added

## Decisions & Rationale

| Decision | Options Considered | Chosen | Rationale |
|----------|-------------------|--------|-----------|
| Template format | Hardcoded JSON vs Markdown | Markdown | ใช้ markdown_to_adf() ที่มีอยู่แล้ว, ง่ายต่อการแก้ไข |
| Argument handling | Inline expansion vs Array | Array | Copilot review - proper shell quoting |
| Dependency links | Manual vs Auto | Both | รองรับ --blocked-by/--blocks flags + manual `link` command |
