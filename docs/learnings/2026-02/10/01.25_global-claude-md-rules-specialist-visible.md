## Key Insight

> `~/.claude/CLAUDE.md` เป็นจุดเดียวที่ทุก Claude instance (รวม specialist) อ่านได้ — ใส่ rules สำคัญไว้ที่นี่

## What We Learned

- Knowledge ใน `claude-km-skill/docs/` specialist เข้าไม่ถึง เพราะทำงานคนละ workspace
- Specialist เห็นแค่ 3 แหล่ง: `~/.claude/CLAUDE.md` (global), `{workspace}/.claude/rules/`, `{workspace}/CLAUDE.md`
- ย้าย rules สำคัญจาก knowledge base มารวมใน global CLAUDE.md เพื่อให้ทุก instance ใช้ร่วมกัน
- Rules ที่เพิ่ม: Git Workflow, Shell Safety, GitHub API patterns

## Gotchas & Warnings

- อย่าใส่ rules มากเกินไปใน global CLAUDE.md — เฉพาะข้อห้ามและ conventions ที่สำคัญจริงๆ
- Project-specific rules ควรอยู่ใน `{project}/CLAUDE.md` ไม่ใช่ global
